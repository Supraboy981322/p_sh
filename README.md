# p_sh; a *very* basic, lightweight shell


>[!NOTE]
>I don't intend to make this fully Posix shell-script compatible; however basic things like inline for-loops and globbing are planned

---

Why make this?
I am actively (at the time of writing this) frustrated with the current shell options.
In my personal opinion (and admittedly limited time trying other shells):
    - zsh is too opinionated and feels clunky
    - I don't like that fish comes with everything pre-enabled (so I never bothered to try it)
    - Bash is lacking some features (without plugins which're likely written in slow shell code) I found myself enjoying in other shells.

---

Why not fork a shell?
To write C, FROM HOW MANY DECADES AGO? (I'd rather spend my freetime writing my own from scratch in Zig)
